1. Introduction

What is Vision Auto?

Vision Auto is a desktop application for automating Android devices using image recognition technology. It enables users to create automation scenarios without requiring programming knowledge or access to the internal structure of apps.

Key Features

Image-Based Automation

Find and interact with UI elements by visual appearance, not internal identifiers.

Real-time Screen Mirroring

See exactly what's happening on your device in real-time.

Conditional Logic

Create complex scenarios with IF/ELSE branching based on screen content.

No Root Required

Works with standard Android devices using only USB debugging.

Use Cases

  • Quality Assurance Testing: Automate repetitive test scenarios
  • Device Configuration: Set up multiple devices with consistent settings
  • App Demonstrations: Create reproducible demo sequences
  • Data Collection: Automate data extraction from visual interfaces

2. Installation

Prerequisites

Requirement Details
Operating System macOS 12+, Windows 10+, or Linux (Ubuntu 20.04+)
Node.js Version 18.0 or later
ADB Android Debug Bridge (any recent version)
RAM 4 GB minimum, 8 GB recommended

Installing ADB

macOS (using Homebrew)

brew install android-platform-tools

Windows

  1. Download Android SDK Platform Tools from the official Android website
  2. Extract the ZIP file to a folder (e.g., C:\platform-tools)
  3. Add the folder path to your system's PATH environment variable

Linux (Debian/Ubuntu)

sudo apt-get update
sudo apt-get install android-tools-adb

Verify Installation

adb version

Installing Vision Auto

Clone the Repository

git clone https://github.com/your-username/vision-auto.git
cd vision-auto

Install Dependencies

npm install

Start the Application

npm start

3. Device Setup

Enabling Developer Options

Open Settings

Navigate to Settings on your Android device.

Find Build Number

Go to "About Phone" and locate "Build Number".

Tap Seven Times

Tap on "Build Number" seven times. You will see a message confirming Developer Options are enabled.

Enabling USB Debugging

Open Developer Options

Go to Settings > Developer Options (or Settings > System > Developer Options).

Enable USB Debugging

Find "USB debugging" and toggle it ON.

Connect Device

Connect your device to the computer via USB cable.

Accept Authorization

On your device, accept the "Allow USB debugging?" prompt. Check "Always allow" for convenience.

Tip: Stay Awake Setting

Enable "Stay awake" in Developer Options to prevent the screen from turning off while the device is connected. This is useful during long automation sessions.

Verifying Connection

adb devices

Expected output:

List of devices attached
XXXXXXXXXX    device
If the device shows "unauthorized"

Disconnect and reconnect the USB cable, then accept the authorization prompt on your device. If no prompt appears, try running adb kill-server and reconnecting.

5. Action Types

Basic Actions

Action Description Parameters
Tap Single touch at specified coordinates X position, Y position
Long Press Extended touch at specified coordinates X position, Y position, Duration (ms)
Swipe Slide from one point to another Start X/Y, End X/Y, Duration (ms)
Drag Touch and hold while moving Start X/Y, End X/Y, Duration (ms)
Text Input Enter text on the focused field Text string
Wait Pause execution for specified duration Duration (ms)

Hardware Key Actions

Action Description
Home Navigate to the home screen
Back Go back or close the current dialog
Recent Apps Open the recent apps view
Volume Up/Down Adjust device volume
Power Toggle screen on/off

Smart Actions

Action Description Parameters
Image Match Search for an image on the screen Template image, Threshold (%), Timeout (ms)
Tap Matched Tap the location of the last successful match None

Control Flow Actions

Action Description
IF Execute contained actions if the condition (image match) is true
ELSE IF Alternative condition check if previous IF was false
ELSE Execute if all previous conditions were false
WHILE Repeat contained actions while the condition is true
LOOP Repeat contained actions a fixed number of times

Result Actions

Action Description
Success Mark the scenario as passed and stop execution
Fail Mark the scenario as failed and stop execution
Skip Skip remaining actions (neutral result)

6. Creating Scenarios

Capturing Image Regions

Enable Crop Mode

Click the "Crop Mode" button in the toolbar. The button will change color to indicate it's active.

Select Region

Click and drag on the screen preview to select the target region. A selection box will appear as you drag.

Configure Action

A dialog will appear with the captured image. Set the match threshold and the action to perform when matched.

Add to Scenario

Click "Add Action" to add the image match action to your scenario.

Best Practices for Image Selection

Good Selections

  • Unique visual elements
  • Include surrounding context
  • Distinct colors or patterns
  • Static elements

Avoid

  • Regions smaller than 20x20 pixels
  • Animated or changing content
  • Elements that appear multiple times
  • Pure white or solid color areas

Setting Match Thresholds

Threshold Range Recommended Use
95-100% Pixel-perfect matches, static content that never changes
85-95% Standard UI elements with minor variations (recommended starting point)
75-85% Dynamic content with consistent layout
60-75% Significant variation expected (use with caution, may cause false positives)

Building Conditional Logic

Example: Login Flow with Error Handling

IF [Login Button visible]
    Tap matched location
    Wait 3000ms
    IF [Home Screen visible]
        Success
    ELSE IF [Error Message visible]
        Tap [Retry Button]
    ELSE
        Fail
    ENDIF
ENDIF

Example: Scroll Until Found

WHILE [Target Item NOT visible]
    Swipe up
    Wait 500ms
ENDWHILE
Tap matched location

Saving and Loading Scenarios

Scenarios are saved as JSON files that can be shared and reused across different computers.

  • Save: Click "Save Scenario" and enter a descriptive name
  • Load: Click "Load Scenario" and select the file
  • Export: Use export function for sharing with others

7. Scenario Execution

Running a Scenario

Verify Connection

Ensure your device is connected and visible in the device panel.

Check Actions

Review the action list to confirm all actions are correctly configured.

Start Execution

Click the "Play" button to begin execution.

Monitor Progress

Watch the screen preview and log panel for execution progress and results.

Execution States

State Description
Idle Ready to execute
Running Scenario is executing
Paused Execution is paused (if applicable)
Completed Scenario finished successfully
Failed Scenario encountered an error
Stopped User stopped the execution

Understanding Results

After execution, the log panel displays detailed results including:

  • Each action executed and its duration
  • Image match scores and results
  • Any errors or warnings encountered
  • Overall scenario result (Pass/Fail)
Tip: AI Analysis

If a scenario fails, you can use the AI Analysis feature to get recommendations for fixing the issue. The analysis considers match scores, device logs, and action history to provide actionable suggestions.

8. Troubleshooting

Device Not Detected

Problem

Device does not appear in the device dropdown.

Solutions:

  1. Verify the USB cable is data-capable (not charge-only)
  2. Check USB debugging is enabled on the device
  3. Accept the "Allow USB debugging" prompt on the device
  4. Restart the ADB server: 
    adb kill-server
adb start-server
adb devices
  5. Try a different USB port

Device Shows "Unauthorized"

Problem

Device appears in the list but shows "unauthorized" status.

Solutions:

  1. Disconnect and reconnect the USB cable
  2. Look for the authorization prompt on the device screen
  3. Check "Always allow from this computer"
  4. If the prompt doesn't appear: 
    adb kill-server
rm ~/.android/adbkey*    # macOS/Linux
adb start-server
  5. Reconnect and accept the prompt

Screen Capture Returns Black Image

Problem

Screen preview shows black or blank image.

Solutions:

  1. Ensure the device screen is unlocked
  2. Check if the current app has a secure flag (banking apps often do)
  3. Disable any screen overlay apps
  4. Try capturing from a different app (e.g., home screen)
  5. Restart the Vision Auto application

Image Matching Always Fails

Problem

Match never succeeds even when the target is visible.

Solutions:

  1. Lower the threshold (try 75-80%)
  2. Recapture the template from the current screen
  3. Ensure device resolution hasn't changed
  4. Avoid capturing animated elements
  5. Select a larger, more distinctive region

Slow Screen Streaming

Problem

Screen preview updates slowly or lags.

Cause Solution
USB 2.0 Use a USB 3.0 port
High resolution Lower device resolution in display settings
CPU load Close other applications
Wireless connection Use USB for better performance

Application Crashes

Problem

Application crashes on startup or during use.

Solutions:

  1. Delete node_modules and reinstall: 
    rm -rf node_modules
npm install
  2. Check Node.js version (18+ required)
  3. Run in debug mode: npm run dev
  4. Check the console for error messages
