1. 소개

Vision Auto란?

Vision Auto는 이미지 인식 기술을 사용하여 Android 디바이스를 자동화하는 데스크톱 애플리케이션입니다. 프로그래밍 지식이나 앱의 내부 구조에 대한 접근 없이도 자동화 시나리오를 생성할 수 있습니다.

주요 기능

이미지 기반 자동화

내부 식별자가 아닌 시각적 모양으로 UI 요소를 찾고 상호작용합니다.

실시간 화면 미러링

디바이스에서 일어나는 일을 실시간으로 정확히 볼 수 있습니다.

조건부 로직

화면 콘텐츠를 기반으로 IF/ELSE 분기가 있는 복잡한 시나리오를 생성합니다.

루팅 불필요

USB 디버깅만 사용하여 표준 Android 디바이스에서 작동합니다.

활용 사례

  • 품질 보증 테스트: 반복적인 테스트 시나리오 자동화
  • 디바이스 설정: 일관된 설정으로 여러 디바이스 구성
  • 앱 데모: 재현 가능한 데모 시퀀스 생성
  • 데이터 수집: 시각적 인터페이스에서 데이터 추출 자동화

2. 설치

사전 요구사항

요구사항 세부 사항
운영 체제 macOS 12+, Windows 10+, 또는 Linux (Ubuntu 20.04+)
Node.js 버전 18.0 이상
ADB Android Debug Bridge (최신 버전)
RAM 최소 4GB, 권장 8GB

ADB 설치

macOS (Homebrew 사용)

brew install android-platform-tools

Windows

  1. 공식 Android 웹사이트에서 Android SDK Platform Tools 다운로드
  2. 폴더에 ZIP 파일 압축 해제 (예: C:\platform-tools)
  3. 시스템 PATH 환경 변수에 폴더 경로 추가

Linux (Debian/Ubuntu)

sudo apt-get update
sudo apt-get install android-tools-adb

설치 확인

adb version

Vision Auto 설치

저장소 복제

git clone https://github.com/guenoh/AVANT.git
cd AVANT

의존성 설치

npm install

애플리케이션 시작

npm start

3. 디바이스 설정

개발자 옵션 활성화

설정 열기

Android 디바이스에서 설정으로 이동합니다.

빌드 번호 찾기

"휴대전화 정보"로 이동하여 "빌드 번호"를 찾습니다.

7번 탭하기

"빌드 번호"를 7번 탭합니다. 개발자 옵션이 활성화되었다는 메시지가 표시됩니다.

USB 디버깅 활성화

개발자 옵션 열기

설정 > 개발자 옵션 (또는 설정 > 시스템 > 개발자 옵션)으로 이동합니다.

USB 디버깅 활성화

"USB 디버깅"을 찾아 켭니다.

디바이스 연결

USB 케이블을 통해 디바이스를 컴퓨터에 연결합니다.

인증 수락

디바이스에서 "USB 디버깅을 허용하시겠습니까?" 프롬프트를 수락합니다. 편의를 위해 "항상 허용"을 체크합니다.

팁: 화면 켜짐 유지 설정

개발자 옵션에서 "화면 켜짐 유지"를 활성화하면 디바이스가 연결된 동안 화면이 꺼지지 않습니다. 긴 자동화 세션에 유용합니다.

연결 확인

adb devices

예상 출력:

List of devices attached
XXXXXXXXXX    device
디바이스가 "unauthorized"로 표시되는 경우

USB 케이블을 분리했다가 다시 연결한 다음 디바이스에서 인증 프롬프트를 수락합니다. 프롬프트가 나타나지 않으면 adb kill-server를 실행하고 다시 연결해 보세요.

6. 시나리오 생성

이미지 영역 캡처

크롭 모드 활성화

툴바에서 "크롭 모드" 버튼을 클릭합니다. 버튼 색상이 변경되어 활성화를 나타냅니다.

영역 선택

화면 미리보기에서 클릭하고 드래그하여 대상 영역을 선택합니다. 드래그할 때 선택 상자가 나타납니다.

액션 설정

캡처된 이미지와 함께 다이얼로그가 나타납니다. 매칭 임계값과 매칭 시 수행할 액션을 설정합니다.

시나리오에 추가

"액션 추가"를 클릭하여 시나리오에 이미지 매칭 액션을 추가합니다.

이미지 선택 모범 사례

좋은 선택

  • 고유한 시각적 요소
  • 주변 컨텍스트 포함
  • 뚜렷한 색상 또는 패턴
  • 정적 요소

피해야 할 것

  • 20x20 픽셀보다 작은 영역
  • 애니메이션되거나 변경되는 콘텐츠
  • 여러 번 나타나는 요소
  • 순수 흰색 또는 단색 영역

매칭 임계값 설정

임계값 범위 권장 용도
95-100% 픽셀 완벽 매칭, 절대 변하지 않는 정적 콘텐츠
85-95% 약간의 변형이 있는 표준 UI 요소 (권장 시작점)
75-85% 일관된 레이아웃의 동적 콘텐츠
60-75% 상당한 변형 예상 (주의해서 사용, 오탐지 발생 가능)

조건부 로직 구축

예제: 오류 처리가 있는 로그인 흐름

IF [로그인 버튼 표시됨]
    매칭된 위치 탭
    3000ms 대기
    IF [홈 화면 표시됨]
        성공
    ELSE IF [에러 메시지 표시됨]
        [재시도 버튼] 탭
    ELSE
        실패
    ENDIF
ENDIF

예제: 찾을 때까지 스크롤

WHILE [대상 항목 표시 안됨]
    위로 스와이프
    500ms 대기
ENDWHILE
매칭된 위치 탭

시나리오 저장 및 불러오기

시나리오는 다른 컴퓨터에서 공유하고 재사용할 수 있는 JSON 파일로 저장됩니다.

  • 저장: "시나리오 저장"을 클릭하고 설명적인 이름 입력
  • 불러오기: "시나리오 불러오기"를 클릭하고 파일 선택
  • 내보내기: 다른 사람과 공유하기 위해 내보내기 기능 사용

7. 시나리오 실행

시나리오 실행하기

연결 확인

디바이스가 연결되어 있고 디바이스 패널에 표시되는지 확인합니다.

액션 확인

액션 목록을 검토하여 모든 액션이 올바르게 설정되었는지 확인합니다.

실행 시작

"재생" 버튼을 클릭하여 실행을 시작합니다.

진행 상황 모니터링

화면 미리보기와 로그 패널에서 실행 진행 상황과 결과를 확인합니다.

실행 상태

상태 설명
대기 중 실행 준비 완료
실행 중 시나리오 실행 중
일시 중지됨 실행이 일시 중지됨 (해당하는 경우)
완료됨 시나리오가 성공적으로 완료됨
실패함 시나리오에서 오류 발생
중지됨 사용자가 실행을 중지함

결과 이해하기

실행 후 로그 패널에 다음을 포함한 상세 결과가 표시됩니다:

  • 실행된 각 액션과 소요 시간
  • 이미지 매칭 점수 및 결과
  • 발생한 오류 또는 경고
  • 전체 시나리오 결과 (통과/실패)
팁: AI 분석

시나리오가 실패하면 AI 분석 기능을 사용하여 문제 해결을 위한 권장 사항을 받을 수 있습니다. 분석은 매칭 점수, 디바이스 로그, 액션 이력을 고려하여 실행 가능한 제안을 제공합니다.

8. 문제 해결

디바이스가 감지되지 않음

문제

디바이스가 디바이스 드롭다운에 나타나지 않습니다.

해결 방법:

  1. USB 케이블이 데이터 전송 가능한지 확인 (충전 전용이 아닌지)
  2. 디바이스에서 USB 디버깅이 활성화되어 있는지 확인
  3. 디바이스에서 "USB 디버깅 허용" 프롬프트 수락
  4. ADB 서버 재시작: 
    adb kill-server
adb start-server
adb devices
  5. 다른 USB 포트 시도

디바이스가 "Unauthorized" 표시

문제

디바이스가 목록에 나타나지만 "unauthorized" 상태로 표시됩니다.

해결 방법:

  1. USB 케이블 분리 후 재연결
  2. 디바이스 화면에서 인증 프롬프트 확인
  3. "이 컴퓨터에서 항상 허용" 체크
  4. 프롬프트가 나타나지 않으면: 
    adb kill-server
rm ~/.android/adbkey*    # macOS/Linux
adb start-server
  5. 재연결하고 프롬프트 수락

화면 캡처가 검은색 이미지 반환

문제

화면 미리보기가 검은색 또는 빈 이미지를 표시합니다.

해결 방법:

  1. 디바이스 화면이 잠금 해제되어 있는지 확인
  2. 현재 앱에 보안 플래그가 있는지 확인 (뱅킹 앱에서 자주 발생)
  3. 화면 오버레이 앱 비활성화
  4. 다른 앱(예: 홈 화면)에서 캡처 시도
  5. Vision Auto 애플리케이션 재시작

이미지 매칭이 항상 실패

문제

대상이 보이는데도 매칭이 성공하지 않습니다.

해결 방법:

  1. 임계값 낮추기 (75-80% 시도)
  2. 현재 화면에서 템플릿 다시 캡처
  3. 디바이스 해상도가 변경되지 않았는지 확인
  4. 애니메이션 요소 캡처 피하기
  5. 더 크고 뚜렷한 영역 선택

느린 화면 스트리밍

문제

화면 미리보기가 느리게 업데이트되거나 지연됩니다.

원인 해결 방법
USB 2.0 USB 3.0 포트 사용
높은 해상도 디스플레이 설정에서 디바이스 해상도 낮추기
CPU 부하 다른 애플리케이션 종료
무선 연결 더 나은 성능을 위해 USB 사용

애플리케이션 충돌

문제

애플리케이션이 시작 시 또는 사용 중 충돌합니다.

해결 방법:

  1. node_modules 삭제 후 재설치: 
    rm -rf node_modules
npm install
  2. Node.js 버전 확인 (18+ 필요)
  3. 디버그 모드로 실행: npm run dev
  4. 콘솔에서 오류 메시지 확인
