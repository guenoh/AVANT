# Vision Auto v2 - 데모 가이드

## 프로젝트 개요

Vision Auto v2는 Android 디바이스 자동화 도구로, 깔끔한 아키텍처와 모던한 UI를 갖춘 Electron 애플리케이션입니다.

## 주요 특징

- **클린 아키텍처**: Main/Renderer/Shared 레이어 분리
- **서비스 기반 설계**: 모듈화된 서비스 구조
- **반응형 상태 관리**: 커스텀 Store 패턴 구현
- **디자인 시스템**: CSS 변수 기반 일관된 UI
- **타입 안전성**: JSDoc 타입 정의

## 기능 시연

### 1. 디바이스 연결
- USB 디버깅이 활성화된 Android 디바이스 연결
- "디바이스 검색" 버튼 클릭
- 디바이스 목록에서 선택
- 디바이스 정보 확인 (모델, 버전, 해상도)

### 2. 화면 미러링
- 화면 탭으로 이동
- "스트리밍 시작" 버튼 클릭
- 실시간 화면 미러링 확인
- 품질과 FPS 조절 가능
- 스크린샷 및 녹화 기능

### 3. 액션 빌더
- 액션 탭으로 이동
- 5가지 액션 타입 지원:
  - **탭 (T)**: 화면 터치
  - **스와이프 (S)**: 드래그 동작
  - **입력 (I)**: 텍스트 입력
  - **대기 (W)**: 지정 시간 대기
  - **키 (K)**: 하드웨어 키 입력
- 액션 파라미터 설정
- 액션 리스트 관리

### 4. 매크로 관리
- 매크로 탭으로 이동
- 새 매크로 생성
- 액션 시퀀스 구성
- 매크로 저장/불러오기
- 매크로 실행 및 녹화

### 5. 설정
- 일반: 언어, 테마, 자동 시작
- 디바이스: 자동 연결, ADB 경로
- 화면: 스트림 품질, 저장 위치
- 매크로: 자동 저장, 백업 설정
- 고급: 디버그 모드, 로그 레벨

## 아키텍처 구조

```
src/
├── main/           # 메인 프로세스
│   ├── main.js     # 앱 엔트리 포인트
│   └── services/   # 백엔드 서비스
│       ├── device.service.js    # ADB 디바이스 제어
│       ├── screen.service.js    # 화면 캡처/스트리밍
│       ├── action.service.js    # 액션 실행
│       ├── macro.service.js     # 매크로 관리
│       ├── settings.service.js  # 설정 관리
│       └── logger.service.js    # 로깅
│
├── renderer/       # 렌더러 프로세스
│   ├── index.html  # UI 레이아웃
│   ├── renderer.js # UI 컨트롤러
│   ├── preload.js  # IPC 브리지
│   └── styles/     # CSS 스타일
│       ├── base.css        # 기본 스타일
│       ├── components.css  # 컴포넌트
│       └── layout.css      # 레이아웃
│
└── shared/         # 공유 모듈
    ├── constants.js # 상수 정의
    ├── store.js     # 상태 관리
    └── utils.js     # 유틸리티
```

## 기술 스택

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Backend**: Node.js, Electron
- **Build**: npm scripts
- **Testing**: 테스트 프레임워크 준비 중
- **CI/CD**: GitHub Actions 준비 중

## 실행 방법

```bash
# 의존성 설치
npm install

# 앱 실행
npm start

# 개발 모드
npm run dev
```

## 문제 해결

### Electron 앱이 실행되지 않는 경우
```bash
# 백그라운드 프로세스 정리
pkill -f electron

# 다시 실행
npm start
```

### ADB 연결 문제
1. USB 디버깅 활성화 확인
2. ADB 드라이버 설치 확인
3. `adb devices` 명령으로 연결 확인
